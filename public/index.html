Course Recommender App

A web app which allows users to train models and then use them to generate recommendations based on courses they have completed.

For each model, the user can select their own hyper parameters to tune their model. This allows for greater customization and also comparison for model performance. The models were created using TensorFlow, Keras surprise libraries. The user has the option to train and use a new model or just use a saved model for predictions.

Most models were relatively straight forward to implement. However when it came to the NeuralNetwork model, it was hard to use this for predictions due to the cold start problem. To solve this, a user profile was created first and then predictions were made. This method was prefered to avoid re-training the model each time, and increase the apps user friendlyness.

In order to improve this app, it may be beneficial for users to get an indication of their trained models performance. Also more models could be added too.

Models that can currently be used include: Course similarity, User Profile, Clustering, Clustering with PCA, K-Neighrest Neighbours, Non-Negative Matrix Factorization, Neural Network, Regression with Embedding and Classification with Embedding.

View this project on GitHub: https://github.com/ronniepiku/CRApp

Sentiment News Trading AI

A trading tool which aids traders in determining market sentiment based on recent news.

In order to collectd data for training, and to make predictions on recent news articles, GNews API is used. Once training data has been collected and pre-processed, it is fed into a CNN and a positive, neutral or negative sentiment is returned.

Currently, the projects works well, however there is much room for improvement. For example, I'm in the process of manually asigning labels to collected data in order to increase the accuraccy of predictions. Currently this is being done programatically which has lead to a positive sentiment being sugested although the news article is unlikely to affect the market. In order to further improve this project, I am also testing different types of models to find which can provide the best performance. This is done by comparing accuracy metrics such as rmse. It would also be nice to collect more data to further improve performance too.

View this project on GitHub: https://github.com/ronniepiku/SANTA

BIR-25 stock selector

This project is being used to manage my personal portfolio which has returned over 20% p/a in the last 3 years.

Firstly, web scrapping techniques are used to collect data on stocks in the FTSE All Stock Index. Each catagory is then ranked into a decile and then assigned a score. The stocks are then sorted by score, and the top 25 stocks in to top decile are selected based on highest 6-month price momentum.

I created this project to reduce a task from something that would take me 7 days to complete, to something that can be reduced to an hours work. This time could be reduced even further, however I faced difficulties avoiding bot detection which is currently done manually. This could be fixed by paying for data instead of scrapping it, but in the interest of maximising profits of the stratagy, a manual method is preffered for now.

Cat/Dog image Classifier

Images of cats and dogs a feed into a sequential TensorFlow model.

Data has already been collected, but is passed through a series of pre-processing steps and is the split into train, test and validation sets.

The model uses a selection of convolutional and 2D-Max Pooling layers which are then flattened and finished off with a relu and softmax layer. Currently the model predicts images correctly 64% of the time.

Performace so far is good, but could be improved. To do so, further tuning should be introduced. For example, it would be worth experimenting with adding or removing layers from the model. Also, hyper perameter tuing should be carried out to determine optimal hyper parameters.

SMS Classifier

NLP is used to determine if an SMS message is spam.

Data has already been collected, but needs to be pre-processed. To do this, a text vectorixation is created. This layer is inserted into a sequential model, which also consists of embedding, bidirectional and reul layers.

After 12 epochs of training, the model achieves a loss of 0.0413 and an accuracy of 0.9899.

Healthcare Cost Predictor

A tool used to help predict healthcare costs using regression analysis.

Data has already been collected but needs to be processed. The steps of processing involves log transformations, followed by one-hot encoding. Processed data is then passed through a sequential model which consists of normalization and dense layers.

After 100 epochs, the model has a loss of 0.9459, a mean absolute error of 0.8110 and a mean squared error of 0.9459. Mean absolute error is the key metric of interest here since it allows for evaluation of errors between a pair of observations.

View this project on GitHub: https://github.com/ronniepiku/Predicting-healthcare-costs

Book Recommendation Engine

Book recommendations are generated given a user input.

Data is loaded in, but needs to be cleaned and processed. First of all, data is put into dataframes. Then users with less than 200 ratings and books with less than 100 ratings are removed. This reduces the impact of anomolous results.

Once processed, the data is fed into a K-Nearest Neighbour model for predictions. The model returns the top 5 book recommendations based on the predicted books distance from the input.
